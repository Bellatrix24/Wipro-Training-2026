# Day 5: Advanced C# Features and SOLID Principles

## Today's Learning Summary
Today was intense! We explored how C# has evolved over the years and learned best practices for writing maintainable code using SOLID principles.

### C# Evolution: 7.0 to 10.0
| Feature (C# 7.0 & 8.0) | Description |
| :--- | :--- |
| **Tuples** | Easily return multiple values from a single method. |
| **Pattern Matching** | Use `switch` or `is` to check types and values cleanly. |
| **Local Functions** | Define a method inside another method. |
| **Async Streams** | Use `await foreach` for asynchronous data streams. |
| **Nullable Ref Types** | Tells the compiler if a reference can be null to avoid crashes. |

**C# 9.0 & 10.0 Features:**
- **Records**: A concise way to create immutable data objects.
- **Top-level statements**: Writing code without the boilerplate `Main` method and `Namespace`.

### Exception Handling
Exception handling is critical because it prevents the program from crashing when something unexpected happens (like a user entering text instead of a number). It allows us to "gracefully" handle errors and keep the user informed.

### SOLID Principles
These are five design principles that help make software more understandable, flexible, and maintainable:
1. **S**ingle Responsibility Principle
2. **O**pen/Closed Principle
3. **L**iskov Substitution Principle
4. **I**nterface Segregation Principle
5. **D**ependency Inversion Principle
